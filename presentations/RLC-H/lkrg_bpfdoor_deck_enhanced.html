01 / THE THREAT

BPFDoor: The Invisible Backdoor

Chinese APT group Red Menshen deployed BPFDoor across thousands of systems globally since 2018.

The attack:

  • Memory-resident malware using Berkeley Packet Filter (BPF)
  • Operates at kernel level, bypassing firewalls
  • No persistent files, no listening ports
  • Self-deletes after installation
  • Detected after 3+ years in documented cases

Standard Linux detection: Requires expert forensic analysis. Most organizations don't have this capability.

02 / THE PROBLEM

Traditional EDR Blind Spots

EDR operates in user space - monitoring applications, processes, and network traffic. But sophisticated threats like BPFDoor live in the kernel.

What EDR misses:

  • Kernel-level packet filters (BPF)
  • Self-deleting executables running from memory
  • Kernel module manipulations
  • Low-level system call hooks
  • Direct firewall rule modifications

The gap: Between kernel space and user space, attackers find room to hide.

04 / THE SOLUTION

LKRG: Linux Kernel Runtime Guard

LKRG provides kernel-level visibility that complements EDR's user-space monitoring.

Process Integrity Monitoring

Detects BPFDoor's signature self-deletion pattern. When a running process deletes its own binary, LKRG flags the integrity violation immediately.

Kernel Integrity Checking

Maintains cryptographic hashes of kernel structures. BPF socket filter installation triggers validation, catching unauthorized modifications.

Usermodehelper Restrictions

Blocks BPFDoor's iptables manipulation by restricting which programs can modify firewall rules from kernel space.

Exploit Detection

Monitors for privilege escalation attempts in real-time, catching the credential tampering BPFDoor uses to gain root access.
